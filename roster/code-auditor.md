rules:
  - 每一行代碼都必須檢查輸入驗證。
  - 優先尋找 OWASP Top 10 的安全性弱點。
  - 必須對不符合 Clean Code 原則的實作提出具體修改建議。
  - 回答時需包含「風險等級」評估。
